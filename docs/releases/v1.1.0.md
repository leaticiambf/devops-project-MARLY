# Mavigo v1.1.0 Release Notes

Release date: 2026-04-22

Tag target: `23f8b6f96dfe100f086c8096eafc3e4c0cd69583`

## Highlights

- Added tourist mode with restaurant suggestions near the user's journey.
- Added a Yelp-backed tourism suggestion backend service and API.
- Introduced the `maview/` Next.js App Router frontend.
- Split local/runtime architecture into a Spring Boot backend and a separate frontend origin.
- Added frontend auth, journey planning, tasks, tourism, and eco-score pages.
- Added frontend test coverage with Vitest and Playwright.

## Backend

- Added `TourismController`, `TourismSuggestionService`, and related DTOs.
- Added ownership checks for user and journey access.
- Updated journey planning request/command DTOs.
- Improved Google Tasks, disruption, eco-score, journey, and user controller behavior.
- Added H2/Postgres-friendly local environment examples.

## Frontend

- Replaced the legacy Spring-served static UI with the Next.js frontend in `maview/`.
- Added reusable API clients, auth storage, layout components, and app providers.
- Added pages for login, registration, journey planning, tasks, tourism exploration, and eco-score.
- Added client-side JWT session restoration via `localStorage`.

## Documentation

- Updated the repository README for the split frontend/backend topology.
- Added frontend-specific development documentation in `maview/README.md`.
- Updated technical documentation for the new application structure.

## Compatibility Notes

- Browser traffic should target the frontend origin.
- The frontend rewrites API and OAuth paths to the configured backend origin.
- Local development expects the backend on `http://localhost:8080` and the frontend on `http://localhost:3000`.
